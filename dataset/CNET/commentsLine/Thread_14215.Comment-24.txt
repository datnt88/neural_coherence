__[ in portuguese (brazil) ]__________________________ pessoal, passei por problema parecido e por isso mesmo acabei caindo nesta pgina de discusso.
como sofri, gostaria de compartilhar minha experincia, para que mais gente no perca o tempo que perdi.
__[ cenrio ]__________________________________________ motherboard asus m2npv-mx, com 2 hds &quot;antigos&quot; ide, seagates de 7.2 e 4.3. ambos conectados no canal ide primrio.
o hd de 7.2 como master e o de 4.3 como slave.
leitor de dvd/cd como master do ide secundrio.
tudo funcionando com windows xp e linux __[ problema ]__________________________________________ resolvi colocar um hd seagate 250 sata-2, mas aps instalar o windows xp e ativar este hd como primeiro dispositivo de boot, recebi a mensagem &quot;erro ao carregar sistema operacional&quot; e em alguns momentor &quot;error loading operating system&quot;.
__[ tentativas ]________________________________________ tentei desabilitar o canal ide primrio, tentei carregar os drivers sata_ide fornecidos pela asus (fabricante da placa-me).
no funcionou.
apesar da instalao do windows detectar o hd, sem precisar dos drivers sata, aps instalar data o problema.
tentei no desabilitar o canal ide primrio e no colocar o hd sata como primeiro dispositivo de boot.
funciona, mas s porque existe o hd ide como primrio.
assim, o gerenciador de boot do windows, ntloader, fica no hd primary master e carrega os arquivos e depois carrega o resto do windows no hd sata.
mesmo assim no era isso o que eu queria.
__[ objetivo a ser solucionado ]_______________________ eu queria e quero que funcione o hd sata como primeiro dispositivo de boot e que o windows e o linux o utilizem para gravar seus gerenciadores de boot, ainda quero utilizar os hds ide como backup, swap, memria virtual.
__[ soluo ]__________________________________________ aps vrias pesquisas e vrias tentativas, as dicas passadas me levaram a conseguir resolver o problema e observar algums coisas.
1. transformei as parties dos hds ides em estendidas.
talvez este item seja opcional 2. durante a instalao do windows notei que tanto o hd sata quanto o hd ide primary master, apareciam como id0 no barramento 0 atapi, ou seja, eles &quot;concorriam&quot; para ser o primeiro dispositivo, desta forma o windows queria gravar a mbr em ambos.
3. resolvi isso invertendo a ordem dos dispositivos ide, ou seja, o dvd/cdrom passou a ser instalado no canal ide primrio, sendo o primary master, e colocando os outros hd no canal ide secundrio.
4. reiniciei a instalao do windows e o hd sata passou a ser o primeiro dispositivo, e os outros hds respectivamente, segundo e terceiro dispositivos da listagem.
nota: no precisei pressionar o f6 durante o incio da instalao do windows, com isso no utilizei quaisquer drivers especficos para enxergar o dispositivo sata.
o windows reconheceu o hd sata como estando no barramento atapi/ide.
5. mesmo assim o windows queria gravar informao no mbr do antigo disco de boot do windows, para isso desativei temporariamente no setup este disco, para poder concluir a instalao do windows.
6. conclui a instalao do windows com sucesso, e depois pude iniciar o sistema via hd sata sem problemas.
7. ativei o hd ide desativado no item 5. o sistema continuou iniciando como feito no item 6.
8. aps backup do contedo, reformatei os hds ide para se adequarem s minhas necessidades.
9. pronto, tudo funcionando e pronto para receber o linux.
espero ter ajudado, pois as diversas discusses na internet, inclusive essa colocada por diversos participantes, me ajudaram a chegar na soluo que atendia minha vontade.
